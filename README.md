** Overview**

This project focuses on analyzing Indian railway schedule data to uncover meaningful patterns, trends, and operational insights.
It was developed as part of a Data Analytics Internship task, using Python for data cleaning, exploration, and visualization.

**🎯 Objectives**

Understand the structure of railway schedule data

Identify busiest stations, routes, and operating days

Visualize trends across weekdays, weekends, and service types

Provide data-driven recommendations for operational improvement

**🧰 Tools & Technologies**

Python

Libraries: Pandas, NumPy, Matplotlib, Seaborn, Plotly, NetworkX

Visualization: Bar charts, Pie charts, Heatmaps, Network graphs

Environment: Jupyter Notebook / VS Code

**📊 Key Insights**

Hub Concentration: Few major stations handle most train departures (hub-and-spoke pattern)

Weekday Dominance: More trains operate on weekdays than weekends

Top Hub: CST-Mumbai shows highest connectivity

Service Patterns: Express and Superfast trains dominate; Passenger and Mail are evenly distributed

Busy Routes: Top 10 station pairs reveal the densest travel corridors

**🖼️ Visual Evidence**

Train frequency per weekday

Day-type distribution (Weekday, Weekend, All-Days)

Train-type heatmap (Express, Mail, Superfast, etc.)

Network graph for top source stations

Top 10 source station bar chart

Heatmap of top 10 origin-destination pairs

**💡 Recommendations**

Reallocate train capacity on peak weekdays

Enhance infrastructure at major hub stations

Increase frequency on high-traffic routes

Adjust service types by weekday/weekend demand

Extend analysis with passenger load and punctuality data
